Vibe Coding Prompt

Give builders something they can paste straight into Cursor or Claude Code

This prompt is meant to turn a product idea into a Hollow-ready connector draft with action schemas, request templates, install settings, and sample payloads.

Replace the bracketed parts with your product, API docs, and use case. The model should give you a strong first draft, not a final production review.

Copy Prompt

Use this to generate a connector manifest and runtime action definitions. 

You are helping me build a Hollow connector.

I need a clean first draft for a connector called [CONNECTOR NAME] that connects Hollow to [PRODUCT OR API].

Context:
- User job to solve: [WHAT THE USER WANTS TO DO]
- API docs or endpoints: [PASTE DOCS OR URL NOTES]
- Auth model: [api_key, oauth2, webhook, or none]
- Important scopes or permissions: [SCOPES]
- Main actions I need: [ACTION 1], [ACTION 2], [ACTION 3]

Return:
1. A short plain-English summary of what the connector does.
2. A Hollow connector JSON payload with:
   - name
   - slug
   - tagline
   - description
   - category
   - visibility
   - status
   - auth_type
   - scopes
   - actions
   - manifest
3. For each action, include:
   - key
   - name
   - description
   - input_schema
   - output_schema
   - request with method, url, headers, query, and body
4. Use {{settings.*}} for secrets like API keys.
5. Use {{input.*}} for runtime values from the user.
6. Include a sample install payload with settings.
7. Include a sample invoke payload for each action.

Important constraints:
- Keep this manifest-driven. Do not invent arbitrary backend code execution.
- Make the copy clear and non-hypey.
- Prefer simple action names and minimal required scopes.
- If the docs are unclear, call that out and make the safest reasonable assumption.
