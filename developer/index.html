Builder Flow

From connector idea to store listing

The portal should walk builders through the exact path from API concept to a public, installable integration.

01

Define your connection

Start with a Hollow manifest describing auth, scopes, actions, and output schemas.

02

Connect real APIs

Map the connector to OAuth, API keys, or webhooks so Hollow can execute against trusted systems.

03

Publish to the store

Trust Model

The store needs guardrails, not just glossy cards.

Creators move faster when permissions, auth, review states, and revocation are obvious from the start.

Reviewable scopes

Every connector declares the exact data it can read or write before a user installs it.

Manifest-first execution

The safest v1 keeps execution inside Hollow with declared actions instead of arbitrary uploaded code.

Human-readable installs

Users should understand what a connector does, what it touches, and how to revoke it.
